# Accessibility Statement
Marine Corps PEBD Calculator & Basic Pay Comparison by Semper Admin

**Effective Date:** June 2, 2026

---

## Commitment to Accessibility

The Marine Corps PEBD Calculator and its companion Basic Pay Comparison page are committed to ensuring digital accessibility for all users, including those with disabilities. We strive to meet Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards on both pages.

---

## Current Accessibility Features

### Keyboard Navigation
- All interactive elements (buttons, form fields, tables) are accessible via keyboard
- Tab order follows logical document flow
- Enter/Space activation for buttons and checkboxes
- Escape key dismisses modals

### Screen Reader Support
- Semantic HTML structure for navigation
- Form labels properly associated with input fields
- Button and link text is descriptive ("Calculate PEBD" not "Click Here")
- Table headers identify column and row content
- Status messages announced to screen readers

### Visual Accessibility
- Dark and light themes: dark navy is the default; a keyboard-accessible toggle (top right of every page) switches to light parchment, and the preference is remembered on your device
- Sufficient color contrast in BOTH themes: every text/background pair is computed against WCAG AA (minimum 4.5:1 for normal text, 3:1 for large text)
- No color used alone to convey information (creditable/not creditable uses text + color)
- Printing always uses the light theme regardless of the on-screen setting
- Responsive design works on mobile, tablet, and desktop
- Text is resizable (browser zoom, no fixed font sizes)
- Focus indicators visible when tabbing through elements

### Motion
- All animations and transitions respect the operating system's "reduce motion" setting (`prefers-reduced-motion`); with it enabled, transitions are effectively disabled on both pages

### Form Accessibility
- All input fields have associated labels
- Error messages (when validation added) will be linked to fields
- Placeholder text does not replace labels
- Required fields can be identified programmatically

### Print Accessibility
- Print reports maintain semantic structure
- All colors readable in black-and-white print
- No information lost when printed without styling

---

## Known Limitations & Workarounds

### Current Limitations
1. **Dynamic table additions:** Adding periods beyond initial 10 may require workaround
   - Workaround: Contact support or use available 10 periods

2. **Complex date calculations:** Screen reader users may find date input formatting challenging
   - Workaround: Use YYYYMMDD format strictly; verify before calculating

3. **Table data density:** Large service period tables may be overwhelming on small screens
   - Workaround: Zoom to 125-150% for easier reading

4. **Print report complexity:** Generated reports contain dense tables
   - Workaround: Export to PDF and review on-screen before printing

## How to Use with Assistive Technology

### NVDA (Free Windows Screen Reader)
1. Download and install NVDA
2. Open this Tool in Firefox or Chrome
3. Navigate with arrow keys; activate with Enter
4. NVDA will announce all form labels and table headers

### JAWS (Licensed Screen Reader)
- Full compatibility expected; tested to WCAG AA
- Table navigation modes supported

### Voice Control (Windows, macOS, iOS)
- All buttons and links can be activated by voice
- Say "Click [button name]" to activate
- Speak form field names to navigate

### Browser Zoom
- Press Ctrl/Cmd + Plus to zoom (increase to 150%+ if needed)
- Layout remains readable up to 200% zoom

## Feedback & Accessibility Issues

If you encounter accessibility barriers:

1. **Describe the issue:** What task were you trying to complete?
2. **Provide your setup:** Browser, OS, assistive technology used
3. **Report it:** Use the Feedback button on any page, or the form at https://forms.osi.apps.mil/r/k5QWzJDL9P (do not include personal information from official records)

We are committed to resolving accessibility issues within 5 business days.
